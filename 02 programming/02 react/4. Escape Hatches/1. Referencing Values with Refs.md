Если вы хотите, чтобы компонент "запомнил" некоторую информацию, но вы не хотите, чтобы эта информация [вызывала](https://react.dev/learn/render-and-commit) [новые рендеры](https://react.dev/learn/render-and-commit), вы можете использовать _ссылку_.
## Adding a ref to your component [](https://react.dev/learn/referencing-values-with-refs#adding-a-ref-to-your-component "Link for Adding a ref to your component")
You can add a ref to your component by importing the `useRef` Hook from React
![[Pasted image 20240527212835.png]]
Inside your component, call the `useRef` Hook and pass the initial value that you want to reference as the only argument. For example, here is a ref to the value `0`:
![[Pasted image 20240527212853.png]]
`useRef`возвращает такой объект:
![[Pasted image 20240527212904.png]]
Вы можете получить доступ к текущему значению этого ref через свойство `ref.current`. Это значение намеренно меняется, что означает, что вы можете читать и писать в него. Это как секретный карман вашего компонента, который React не отслеживает. (Это то, что делает его "вылупляемым" из одностороннего потока данных React - подробнее об этом ниже!)
Here, a button will increment `ref.current` on every click:
```js
import { useRef } from 'react';

export default function Counter() {
  let ref = useRef(0);

  function handleClick() {
    ref.current = ref.current + 1;
    alert('You clicked ' + ref.current + ' times!');
  }

  return (
    <button onClick={handleClick}>
      Click me!
    </button>
  );
}
```
Ссылка указывает на число, но, как и на состояние, вы можете указывать на что угодно: строку, объект или даже функцию. В отличие от состояния, ref — это простой объект JavaScript с текущим свойством, которое вы можете читать и изменять.
Обратите внимание, что **компонент не повторяется с каждым шагом.** Как и состояние, ссылки сохраняются React между повторными рендерами. Однако установка состояния повторно рендерит компонент. Изменение ссылки не делает!
## Example: building a stopwatch [](https://react.dev/learn/referencing-values-with-refs#example-building-a-stopwatch "Link for Example: building a stopwatch")
Вы можете объединить ссылки и состояние в одном компоненте. Например, давайте сделаем секундомер, который пользователь может запускать или останавливать нажатием кнопки. Чтобы отобразить, сколько времени прошло с момента нажатия пользователем «Старт», вам необходимо будет отслеживать, когда была нажата кнопка «Старт» и какое сейчас время. Эта информация используется для рендеринга, поэтому вы сохраните ее в следующем состоянии:
![[Pasted image 20240527213136.png]]
Когда пользователь нажмет "Пуск", вы будете использовать [`setInterval`](https://developer.mozilla.org/docs/Web/API/setInterval), чтобы обновлять время каждые 10 миллисекунд:
```js
import { useState } from 'react';

export default function Stopwatch() {
  const [startTime, setStartTime] = useState(null);
  const [now, setNow] = useState(null);

  function handleStart() {
    // Start counting.
    setStartTime(Date.now());
    setNow(Date.now());

    setInterval(() => {
      // Update the current time every 10ms.
      setNow(Date.now());
    }, 10);
  }

  let secondsPassed = 0;
  if (startTime != null && now != null) {
    secondsPassed = (now - startTime) / 1000;
  }

  return (
    <>
      <h1>Time passed: {secondsPassed.toFixed(3)}</h1>
      <button onClick={handleStart}>
        Start
      </button>
    </>
  );
}

```
При нажатии кнопки "Стоп" необходимо отменить существующий интервал, чтобы он перестал обновлять переменную состояния `now`. Вы можете сделать это, вызвав [`clearInterval`](https://developer.mozilla.org/en-US/docs/Web/API/clearInterval), но вам нужно дать ему идентификатор интервала, который ранее был возвращен вызовом `setInterval`, когда пользователь нажал Start. Вам нужно где-то хранить идентификатор интервала. **Поскольку идентификатор интервала не используется для рендеринга, вы можете сохранить его в ссылке:**
```js
import { useState, useRef } from 'react';

export default function Stopwatch() {
  const [startTime, setStartTime] = useState(null);
  const [now, setNow] = useState(null);
  const intervalRef = useRef(null);

  function handleStart() {
    setStartTime(Date.now());
    setNow(Date.now());

    clearInterval(intervalRef.current);
    intervalRef.current = setInterval(() => {
      setNow(Date.now());
    }, 10);
  }

  function handleStop() {
    clearInterval(intervalRef.current);
  }

  let secondsPassed = 0;
  if (startTime != null && now != null) {
    secondsPassed = (now - startTime) / 1000;
  }

  return (
    <>
      <h1>Time passed: {secondsPassed.toFixed(3)}</h1>
      <button onClick={handleStart}>
        Start
      </button>
      <button onClick={handleStop}>
        Stop
      </button>
    </>
  );
}

```
Когда часть информации используется для рендеринга, держите ее в состоянии. Когда часть информации нужна только обработчикам событий, и ее изменение не требует повторного рендеринга, использование ссылки может быть более эффективным.
## Differences between refs and state [](https://react.dev/learn/referencing-values-with-refs#differences-between-refs-and-state "Link for Differences between refs and state")
Возможно, вы думаете, что ссылки кажутся менее "строгими", чем состояние - вы можете мутировать их вместо того, чтобы всегда использовать функцию настройки состояния, например. Но в большинстве случаев вы захотите использовать состояние. Рефы - это "побег", которые вам не понадобятся часто. Вот как сравниваются штат и ссылки:
![[Pasted image 20240527213555.png]]
Вот кнопка счетчика, которая реализована со состоянием:
```js
import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <button onClick={handleClick}>
      You clicked {count} times
    </button>
  );
}

```
Поскольку отображается значение `count`, имеет смысл использовать для него значение состояния. Когда значение счетчика установлено с помощью `setCount()`, React повторно отображает компонент, и экран обновляется, чтобы отразить новый счетчик.
Если бы вы попытались реализовать это с помощью ссылки, React никогда бы не повторно отрендерил компонент, так что вы никогда бы не увидели изменения количества! Посмотрите, как нажатие этой кнопки **не обновляет ее текст**:
```js
import { useRef } from 'react';

export default function Counter() {
  let countRef = useRef(0);

  function handleClick() {
    // This doesn't re-render the component!
    countRef.current = countRef.current + 1;
  }

  return (
    <button onClick={handleClick}>
      You clicked {countRef.current} times
    </button>
  );
}

```

Вот почему чтение ref.current во время рендеринга приводит к ненадежному коду. Если вам это нужно, используйте вместо этого состояние.

#### How does useRef work inside?
Хотя и useState, и useRef предоставляются React, в принципе useRef можно реализовать поверх useState. Вы можете себе представить, что внутри React useRef реализован следующим образом:
![[Pasted image 20240527213910.png]]
Во время первого рендеринга useRef возвращает { current: InitialValue }. Этот объект хранится в React, поэтому при следующем рендеринге будет возвращен тот же объект. Обратите внимание, что в этом примере установщик состояния не используется. В этом нет необходимости, поскольку useRef всегда должен возвращать один и тот же объект!
React предоставляет встроенную версию useRef, поскольку она достаточно распространена на практике. Но вы можете думать об этом как об обычной переменной состояния без установщика. Если вы знакомы с объектно-ориентированным программированием, ссылки могут напомнить вам поля экземпляров, но вместо this.something вы пишете что-тоRef.current.
## When to use refs [](https://react.dev/learn/referencing-values-with-refs#when-to-use-refs "Link for When to use refs")
Как правило, вы будете использовать ссылку, когда вашему компоненту нужно "выйти за пределы" React и связаться с внешними API - часто API браузера, который не повлияет на внешний вид компонента. Вот некоторые из этих редких ситуаций:
- Хранение [идентификаторов тайм-аута](https://developer.mozilla.org/docs/Web/API/setTimeout)
- - Хранение и манипулирование [элементами DOM](https://developer.mozilla.org/docs/Web/API/Element), которые мы рассмотрим на [следующей странице](https://react.dev/learn/manipulating-the-dom-with-refs)
- - Хранение других объектов, которые не нужны для расчета JSX.
Если вашему компоненту необходимо сохранить какое-то значение, но это не влияет на логику рендеринга, выберите ссылки.
## Best practices for refs [](https://react.dev/learn/referencing-values-with-refs#best-practices-for-refs "Link for Best practices for refs")
Следование этим принципам сделает ваши компоненты более предсказуемыми:
- **Относитесь к реферам как к эвакуационному люку.** Ссылки полезны при работе с внешними системами или API браузера. Если большая часть вашей логики приложения и потока данных зависит от ссылок, вы можете переосмыслить свой подход.
- Не читайте и не записывайте ref.current во время рендеринга. Если во время рендеринга требуется какая-то информация, используйте вместо нее состояние. Поскольку React не знает, когда изменяется ref.current, даже чтение его во время рендеринга затрудняет прогнозирование поведения вашего компонента. (Единственным исключением является код типа if (!ref.current) ref.current = new Thing(), который устанавливает ссылку только один раз во время первого рендеринга.)
Ограничения состояния React не распространяются на ссылки. Например, состояние действует как [снимок для каждого рендера](https://react.dev/learn/state-as-a-snapshot) и не [обновляется синхронно.](https://react.dev/learn/queueing-a-series-of-state-updates) Но когда вы изменяете текущее значение ссылки, оно сразу же меняется:
![[Pasted image 20240527214511.png]]
Это связано **с тем**, что **сам реф является обычным объектом JavaScript,** и поэтому он ведет себя как таковой.
Вам также не нужно беспокоиться об [избежании мутации](https://react.dev/learn/updating-objects-in-state), когда вы работаете с рефер. До тех пор, пока объект, который вы мутируете, не используется для рендеринга, React не заботится о том, что вы делаете с ссылкой или его содержимым.
## Refs and the DOM [](https://react.dev/learn/referencing-values-with-refs#refs-and-the-dom "Link for Refs and the DOM")
Вы можете указать ссылку на любое значение. Однако наиболее распространенным вариантом использования ссылки является доступ к элементу DOM. Например, это удобно, если вы хотите программно сфокусировать ввод. Когда вы передаете ссылку на атрибут ref в JSX, например \<div ref={myRef}>, React поместит соответствующий элемент DOM в myRef.current. Как только элемент будет удален из DOM, React обновит myRef.current до нулевого значения. Подробнее об этом можно прочитать в разделе «Управление DOM с помощью ссылок».
