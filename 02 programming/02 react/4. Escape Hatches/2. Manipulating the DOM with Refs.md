React автоматически обновляет [DOM](https://developer.mozilla.org/docs/Web/API/Document_Object_Model/Introduction) в соответствии с вашим выводом рендеринга, поэтому вашим компонентам не придется часто манипулировать им. Однако иногда вам может понадобиться доступ к элементам DOM, управляемым React, например, чтобы сфокусировать узел, прокрутить до него или измерить его размер и положение. В React нет встроенного способа сделать эти вещи, поэтому вам понадобится _ссылка_ на узел DOM.
## Getting a ref to the node [](https://react.dev/learn/manipulating-the-dom-with-refs#getting-a-ref-to-the-node "Link for Getting a ref to the node")

Finally, pass your ref as the `ref` attribute to the JSX tag for which you want to get the DOM node:

Хук useRef возвращает объект с единственным свойством, называемым текущим. Первоначально myRef.current будет иметь значение null. Когда React создает узел DOM для этого \<div>, React поместит ссылку на этот узел в myRef.current. Затем вы можете получить доступ к этому узлу DOM из своих обработчиков событий и использовать встроенные API-интерфейсы браузера, определенные на нем.

### Example: Focusing a text input [](https://react.dev/learn/manipulating-the-dom-with-refs#example-focusing-a-text-input "Link for Example: Focusing a text input")
В этом примере нажатие кнопки сфокусирует входные данные:
```js
import { useRef } from 'react';

export default function Form() {
  const inputRef = useRef(null);

  function handleClick() {
    inputRef.current.focus();
  }

  return (
    <>
      <input ref={inputRef} />
      <button onClick={handleClick}>
        Focus the input
      </button>
    </>
  );
}

```
Чтобы реализовать это:
1. Declare `inputRef` with the `useRef` Hook.
2.  Pass it as `<input ref={inputRef}>`. This tells React to **put this `<input>`’s DOM node into `inputRef.current`.**
3.  In the `handleClick` function, read the input DOM node from `inputRef.current` and call [`focus()`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/focus) on it with `inputRef.current.focus()`.
4. 1. Pass the `handleClick` event handler to `<button>` with `onClick`.
Хотя манипуляции с DOM являются наиболее распространенным вариантом использования ссылок, хук useRef можно использовать для хранения других вещей вне React, например идентификаторов таймеров. Как и в случае с состоянием, между рендерингами сохраняются ссылки. Ссылки похожи на переменные состояния, которые не вызывают повторный рендеринг при их установке. Прочтите о ссылках в разделе «Ссылки на значения с помощью ссылок».

#### How to manage a list of refs using a ref callback
В приведенных выше примерах есть предопределенное количество ссылок. Тем не менее, иногда вам может понадобиться ссылка на каждый элемент в списке, и вы не знаете, сколько у вас будет. Что-то вроде этого не **сработает**:
Это связано с тем, что хуки следует вызывать только на верхнем уровне вашего компонента. Вы не можете вызывать useRef в цикле, в условии или внутри вызова map().
Один из возможных способов обойти это - получить одну ссылку на родительский элемент, а затем использовать методы обработки DOM, такие как [`querySelectorAll`](https://developer.mozilla.org/en-US/docs/Web/API/Document/querySelectorAll), чтобы "найти" отдельные дочерние узлы из него. Тем не менее, это хрупко и может сломаться, если ваша структура DOM изменится.
Другим решением является передание **функции атрибуту `ref`.** Это называется [обратным](https://react.dev/reference/react-dom/components/common#ref-callback) [вызовом `ref`.](https://react.dev/reference/react-dom/components/common#ref-callback) React вызовет ваш обратный вызов ref с узлом DOM, когда придет время установить ref, и с `null`, когда придет время очистить его. Это позволяет вам поддерживать свой собственный массив или [карту](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Map) и получать доступ к любому ссылке по его индексу или какому-либо идентификатору.
Этот пример показывает, как вы можете использовать этот подход для прокрутки до произвольного узла в длинном списке:

```js
import { useRef, useState } from "react";

export default function CatFriends() {
  const itemsRef = useRef(null);
  const [catList, setCatList] = useState(setupCatList);

  function scrollToCat(cat) {
    const map = getMap();
    const node = map.get(cat);
    node.scrollIntoView({
      behavior: "smooth",
      block: "nearest",
      inline: "center",
    });
  }

  function getMap() {
    if (!itemsRef.current) {
      // Initialize the Map on first usage.
      itemsRef.current = new Map();
    }
    return itemsRef.current;
  }

  return (
    <>
      <nav>
        <button onClick={() => scrollToCat(catList[0])}>Tom</button>
        <button onClick={() => scrollToCat(catList[5])}>Maru</button>
        <button onClick={() => scrollToCat(catList[9])}>Jellylorum</button>
      </nav>
      <div>
        <ul>
          {catList.map((cat) => (
            <li
              key={cat}
              ref={(node) => {
                const map = getMap();
                if (node) {
                  map.set(cat, node);
                } else {
                  map.delete(cat);
                }
              }}
            >
              <img src={cat} />
            </li>
          ))}
        </ul>
      </div>
    </>
  );
}

function setupCatList() {
  const catList = [];
  for (let i = 0; i < 10; i++) {
    catList.push("https://loremflickr.com/320/240/cat?lock=" + i);
  }

  return catList;
}


```

В этом примере itemsRef не содержит ни одного узла DOM. Вместо этого он содержит карту от идентификатора элемента до узла DOM. (Ссылки могут содержать любые значения!) Обратный вызов ref для каждого элемента списка заботится об обновлении карты:

## Accessing another component’s DOM nodes [](https://react.dev/learn/manipulating-the-dom-with-refs#accessing-another-components-dom-nodes "Link for Accessing another component’s DOM nodes")
Когда вы помещаете ссылку на встроенный компонент, который выводит элемент браузера, например <input />, React установит текущее свойство этой ссылки на соответствующий узел DOM (например, фактический <input /> в браузере).

Однако если вы попытаетесь поместить ссылку на свой собственный компонент, например <MyInput />, по умолчанию вы получите значение null. Вот пример, демонстрирующий это. Обратите внимание, что нажатие кнопки не фокусирует ввод:
```js
import { useRef } from 'react';

function MyInput(props) {
  return <input {...props} />;
}

export default function MyForm() {
  const inputRef = useRef(null);

  function handleClick() {
    inputRef.current.focus();
  }

  return (
    <>
      <MyInput ref={inputRef} />
      <button onClick={handleClick}>
        Focus the input
      </button>
    </>
  );
}

```
Чтобы помочь вам заметить проблему, React также печатает ошибку на консоли:
![[Pasted image 20240527223517.png]]
Это происходит потому, что по умолчанию React не позволяет компоненту получать доступ к узлам DOM других компонентов. Даже не для своих собственных детей! Это преднамеренно. Refs - это эвакуационный люк, который следует использовать экономно. Ручная манипуляция с узлами DOM _другого_ компонента делает ваш код еще более хрупким.
Вместо этого компоненты, которые хотят предоставить доступ к своим узлам DOM, должны согласиться на такое поведение. Компонент может указать, что он «forwardRef» свою ссылку одному из своих дочерних элементов. Вот как MyInput может использовать API frontRef:
Вот как это работает:
1. \<MyInput ref={inputRef} /> сообщает React поместить соответствующий узел DOM в inputRef.current. Однако компонент MyInput должен разрешить это — по умолчанию это не так.
2. Компонент MyInput объявляется с помощью forwardRef. Это позволяет ему получать входную ссылку сверху в качестве второго аргумента ref, который объявляется после реквизита.
3. MyInput сам передает полученную ссылку в \<input> внутри него.

Теперь нажатие кнопки для фокусировки ввода работает:
```js
import { forwardRef, useRef } from 'react';

const MyInput = forwardRef((props, ref) => {
  return <input {...props} ref={ref} />;
});

export default function Form() {
  const inputRef = useRef(null);

  function handleClick() {
    inputRef.current.focus();
  }

  return (
    <>
      <MyInput ref={inputRef} />
      <button onClick={handleClick}>
        Focus the input
      </button>
    </>
  );
}

```
В системах проектирования это общий шаблон для низкоуровневых компонентов, таких как кнопки, входы и т. д., чтобы переслать свои ссылки на узлы DOM. С другой стороны, высокоуровневые компоненты, такие как формы, списки или разделы страниц, обычно не предоставляют свои DOM-узлы, чтобы избежать случайных зависимостей от структуры DOM
#### Exposing a subset of the API with an imperative handle
В приведенном выше примере MyInput предоставляет исходный элемент ввода DOM. Это позволяет родительскому компоненту вызывать на нем focus(). Однако это также позволяет родительскому компоненту делать что-то еще, например изменять стили CSS. В редких случаях вам может потребоваться ограничить предоставляемую функциональность. Вы можете сделать это с помощью useImperativeHandle:
```js
import {
  forwardRef, 
  useRef, 
  useImperativeHandle
} from 'react';

const MyInput = forwardRef((props, ref) => {
  const realInputRef = useRef(null);
  useImperativeHandle(ref, () => ({
    // Only expose focus and nothing else
    focus() {
      realInputRef.current.focus();
    },
  }));
  return <input {...props} ref={realInputRef} />;
});

export default function Form() {
  const inputRef = useRef(null);

  function handleClick() {
    inputRef.current.focus();
  }

  return (
    <>
      <MyInput ref={inputRef} />
      <button onClick={handleClick}>
        Focus the input
      </button>
    </>
  );
}

```
Здесь realInputRef внутри MyInput содержит фактический входной узел DOM. Однако useImperativeHandle инструктирует React предоставить ваш собственный специальный объект в качестве значения ссылки на родительский компонент. Таким образом, inputRef.current внутри компонента Form будет иметь только метод фокуса. В этом случае ref «дескриптор» — это не узел DOM, а пользовательский объект, который вы создаете внутри вызова useImperativeHandle.
## When React attaches the refs [](https://react.dev/learn/manipulating-the-dom-with-refs#when-react-attaches-the-refs "Link for When React attaches the refs")
В React каждое обновление делится на [два этапа](https://react.dev/learn/render-and-commit#step-3-react-commits-changes-to-the-dom):
- Во время **рендеринга** React вызывает ваши компоненты, чтобы выяснить, что должно быть на экране.
- Во время **commit** React применяет изменения к DOM.
В общем, вам не нужен доступ к ссылкам во время рендеринга. Это касается и ссылок, содержащих узлы DOM. Во время первого рендеринга узлы DOM еще не были созданы, поэтому ref.current будет иметь значение null. И во время рендеринга обновлений узлы DOM еще не обновились. Так что читать их пока рано.
React устанавливает ref.current во время commit. Перед обновлением DOM React устанавливает для затронутых значений ref.current значение null. После обновления DOM React немедленно устанавливает их в соответствующие узлы DOM.
**Обычно вы получаете доступ к ссылкам от обработчиков событий.** Если вы хотите сделать что-то с ref, но нет конкретного события, чтобы сделать это, вам может понадобиться эффект. Мы обсудим эффекты на следующих страницах.
#### Flushing state updates synchronously with flushSync [](https://react.dev/learn/manipulating-the-dom-with-refs#flushing-state-updates-synchronously-with-flush-sync "Link for Flushing state updates synchronously with flushSync")
Рассмотрим такой код, который добавляет новую задачу и прокручивает экран вниз до последнего дочернего списка. Обратите внимание, что по какой-то причине он всегда прокручивается до того, что было _непосредственно перед_ последним добавленным:
```js
import { useState, useRef } from 'react';

export default function TodoList() {
  const listRef = useRef(null);
  const [text, setText] = useState('');
  const [todos, setTodos] = useState(
    initialTodos
  );

  function handleAdd() {
    const newTodo = { id: nextId++, text: text };
    setText('');
    setTodos([ ...todos, newTodo]);
    listRef.current.lastChild.scrollIntoView({
      behavior: 'smooth',
      block: 'nearest'
    });
  }

  return (
    <>
      <button onClick={handleAdd}>
        Add
      </button>
      <input
        value={text}
        onChange={e => setText(e.target.value)}
      />
      <ul ref={listRef}>
        {todos.map(todo => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </>
  );
}

let nextId = 0;
let initialTodos = [];
for (let i = 0; i < 20; i++) {
  initialTodos.push({
    id: nextId++,
    text: 'Todo #' + (i + 1)
  });
}

```
Проблема в этих двух строках:
![[Pasted image 20240527225054.png]]
В React обновления состояния ставятся в очередь. Обычно это то, что вам нужно. Однако здесь возникает проблема, поскольку setTodos не обновляет DOM немедленно. Таким образом, когда вы прокручиваете список до последнего элемента, задача еще не добавлена. Именно поэтому прокрутка всегда «отстает» на один пункт.

Чтобы решить эту проблему, вы можете заставить React синхронно обновлять («очистить») DOM. Для этого импортируйте флешсинк из реагирования и оберните обновление состояния в вызов флешсинк:
![[Pasted image 20240527225131.png]]
Это подаст React указано обновлять DOM синхронно сразу после выполнения кода, завернутого в `flushSync`. В результате, последнее дело уже будет в DOM к тому времени, когда вы попытаетесь прокрутить до него:
```js
import { useState, useRef } from 'react';
import { flushSync } from 'react-dom';

export default function TodoList() {
  const listRef = useRef(null);
  const [text, setText] = useState('');
  const [todos, setTodos] = useState(
    initialTodos
  );

  function handleAdd() {
    const newTodo = { id: nextId++, text: text };
    flushSync(() => {
      setText('');
      setTodos([ ...todos, newTodo]);      
    });
    listRef.current.lastChild.scrollIntoView({
      behavior: 'smooth',
      block: 'nearest'
    });
  }

  return (
    <>
      <button onClick={handleAdd}>
        Add
      </button>
      <input
        value={text}
        onChange={e => setText(e.target.value)}
      />
      <ul ref={listRef}>
        {todos.map(todo => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </>
  );
}

let nextId = 0;
let initialTodos = [];
for (let i = 0; i < 20; i++) {
  initialTodos.push({
    id: nextId++,
    text: 'Todo #' + (i + 1)
  });
}

```
## Best practices for DOM manipulation with refs [](https://react.dev/learn/manipulating-the-dom-with-refs#best-practices-for-dom-manipulation-with-refs "Link for Best practices for DOM manipulation with refs")
Рефы - это эвакуационный люк. Вы должны использовать их только тогда, когда вам нужно "выйти за пределы React". Общие примеры этого включают в себя управление фокусом, положением прокрутки или вызов API браузера, которые React не предоставляет.
Если вы придерживаетесь неразрушающих действий, таких как фокусировка и прокрутка, вы не должны сталкиваться с какими-либо проблемами. Однако, если вы попытаетесь **изменить** DOM вручную, вы можете рискнуть конфликтовать с изменениями, которые вносит React.
Чтобы проиллюстрировать эту проблему, этот пример включает приветственное сообщение и две кнопки. Первая кнопка переключает свое присутствие с помощью условного рендеринга и состояния, как вы обычно делаете в React. Вторая кнопка использует API DOM remove() для принудительного удаления объекта из DOM вне контроля React.
Попробуйте несколько раз нажать "Переключить с setState". Сообщение должно исчезнуть и снова появиться. Затем нажмите "Удалить из DOM". Это принудительно удалит его. Наконец, нажмите "Переключить с setState":
```js
import { useState, useRef } from 'react';

export default function Counter() {
  const [show, setShow] = useState(true);
  const ref = useRef(null);

  return (
    <div>
      <button
        onClick={() => {
          setShow(!show);
        }}>
        Toggle with setState
      </button>
      <button
        onClick={() => {
          ref.current.remove();
        }}>
        Remove from the DOM
      </button>
      {show && <p ref={ref}>Hello world</p>}
    </div>
  );
}

```
После того как вы удалили элемент DOM вручную, попытка использовать setState для его повторного отображения приведет к сбою. Это связано с тем, что вы изменили DOM, и React не знает, как продолжать правильно им управлять.
**Избегайте изменения узлов DOM, управляемых React.** Изменение, добавление дочерних элементов или удаление дочерних элементов из элементов, управляемых React, может привести к непоследовательным визуальным результатам или сбоям, как указано выше.
Однако это не значит, что вы вообще этого не можете сделать. Это требует осторожности. Вы можете безопасно изменять части DOM, которые у React нет причин обновлять. Например, если какой-либо \<div> всегда пуст в JSX, у React не будет причин трогать свой дочерний список. Поэтому туда можно безопасно добавлять или удалять элементы вручную.
