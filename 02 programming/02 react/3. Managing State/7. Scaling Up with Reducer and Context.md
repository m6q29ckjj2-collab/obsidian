Редукторы позволяют консолидировать логику обновления состояния компонента. Контекст позволяет передавать информацию в глубине души в другие компоненты. Вы можете объединить редукторы и контекст вместе для управления состоянием сложного экрана.
## Combining a reducer with context [](https://react.dev/learn/scaling-up-with-reducer-and-context#combining-a-reducer-with-context "Link for Combining a reducer with context")
В этом примере [](https://react.dev/learn/extracting-state-logic-into-a-reducer)из [введения в редукторы](https://react.dev/learn/extracting-state-logic-into-a-reducer) состояние управляется редуктором. Функция редуктора содержит всю логику обновления состояния и объявлена в нижней части этого файла:
```js
import { useReducer } from 'react';
import AddTask from './AddTask.js';
import TaskList from './TaskList.js';

export default function TaskApp() {
  const [tasks, dispatch] = useReducer(
    tasksReducer,
    initialTasks
  );

  function handleAddTask(text) {
    dispatch({
      type: 'added',
      id: nextId++,
      text: text,
    });
  }

  function handleChangeTask(task) {
    dispatch({
      type: 'changed',
      task: task
    });
  }

  function handleDeleteTask(taskId) {
    dispatch({
      type: 'deleted',
      id: taskId
    });
  }

  return (
    <>
      <h1>Day off in Kyoto</h1>
      <AddTask
        onAddTask={handleAddTask}
      />
      <TaskList
        tasks={tasks}
        onChangeTask={handleChangeTask}
        onDeleteTask={handleDeleteTask}
      />
    </>
  );
}

function tasksReducer(tasks, action) {
  switch (action.type) {
    case 'added': {
      return [...tasks, {
        id: action.id,
        text: action.text,
        done: false
      }];
    }
    case 'changed': {
      return tasks.map(t => {
        if (t.id === action.task.id) {
          return action.task;
        } else {
          return t;
        }
      });
    }
    case 'deleted': {
      return tasks.filter(t => t.id !== action.id);
    }
    default: {
      throw Error('Unknown action: ' + action.type);
    }
  }
}

let nextId = 3;
const initialTasks = [
  { id: 0, text: 'Philosopher’s Path', done: true },
  { id: 1, text: 'Visit the temple', done: false },
  { id: 2, text: 'Drink matcha', done: false }
];

```
Редьюсер помогает сделать обработчики событий короткими и лаконичными. Однако по мере роста вашего приложения вы можете столкнуться с другой трудностью. В настоящее время состояние задач и функция отправки доступны только в компоненте TaskApp верхнего уровня. Чтобы позволить другим компонентам читать список задач или изменять его, вам необходимо явно передать текущее состояние и обработчики событий, которые его изменяют, в качестве реквизита.
For example, `TaskApp` passes a list of tasks and the event handlers to `TaskList`:

And `TaskList` passes the event handlers to `Task`:

В таком маленьком примере это работает хорошо, но если у вас есть десятки или сотни компонентов в середине, передача всех состояний и функций может быть довольно разочаровывающей!
Вот почему в качестве альтернативы передаче их через реквизиты вы можете поместить в контекст как состояние задач, так и функцию отправки. Таким образом, любой компонент ниже TaskApp в дереве может считывать задачи и выполнять действия без повторяющегося «prop driling».
Вот как вы можете соединить редуктор с контекстом:

1. **Создайте** контекст.
2. **Поместите** состояние и отправку в контекст.
3. **Используйте** контекст в любом месте дерева.
### Step 1: Create the context [](https://react.dev/learn/scaling-up-with-reducer-and-context#step-1-create-the-context "Link for Step 1: Create the context")
he `useReducer` Hook returns the current `tasks` and the `dispatch` function that lets you update them:
Чтобы передать их по дереву, вы [создадите](https://react.dev/learn/passing-data-deeply-with-context#step-2-use-the-context) два отдельных контекста:
- `TasksContext`предоставляет текущий список задач.
- - `TasksDispatchContext`предоставляет функцию, которая позволяет компонентам отправлять действия.
Экспортируйте их из отдельного файла, чтобы впоследствии импортировать их из других файлов:
```js
import { createContext } from 'react';

export const TasksContext = createContext(null);
export const TasksDispatchContext = createContext(null);

```
Здесь вы передаете значение null в качестве значения по умолчанию для обоих контекстов. Фактические значения будут предоставлены компонентом TaskApp.
### Step 2: Put state and dispatch into context [](https://react.dev/learn/scaling-up-with-reducer-and-context#step-2-put-state-and-dispatch-into-context "Link for Step 2: Put state and dispatch into context")
Теперь вы можете импортировать оба контекста в свой компонент TaskApp. Возьмите задачи и диспетчеризацию, возвращаемые useReducer(), и передайте их всему дереву ниже:
![[Pasted image 20240527212029.png]]
На данный момент вы передаете информацию как через реквизит, так и в контексте:
```js
import { useReducer } from 'react';
import AddTask from './AddTask.js';
import TaskList from './TaskList.js';
import { TasksContext, TasksDispatchContext } from './TasksContext.js';

export default function TaskApp() {
  const [tasks, dispatch] = useReducer(
    tasksReducer,
    initialTasks
  );

  function handleAddTask(text) {
    dispatch({
      type: 'added',
      id: nextId++,
      text: text,
    });
  }

  function handleChangeTask(task) {
    dispatch({
      type: 'changed',
      task: task
    });
  }

  function handleDeleteTask(taskId) {
    dispatch({
      type: 'deleted',
      id: taskId
    });
  }

  return (
    <TasksContext.Provider value={tasks}>
      <TasksDispatchContext.Provider value={dispatch}>
        <h1>Day off in Kyoto</h1>
        <AddTask
          onAddTask={handleAddTask}
        />
        <TaskList
          tasks={tasks}
          onChangeTask={handleChangeTask}
          onDeleteTask={handleDeleteTask}
        />
      </TasksDispatchContext.Provider>
    </TasksContext.Provider>
  );
}

function tasksReducer(tasks, action) {
  switch (action.type) {
    case 'added': {
      return [...tasks, {
        id: action.id,
        text: action.text,
        done: false
      }];
    }
    case 'changed': {
      return tasks.map(t => {
        if (t.id === action.task.id) {
          return action.task;
        } else {
          return t;
        }
      });
    }
    case 'deleted': {
      return tasks.filter(t => t.id !== action.id);
    }
    default: {
      throw Error('Unknown action: ' + action.type);
    }
  }
}

let nextId = 3;
const initialTasks = [
  { id: 0, text: 'Philosopher’s Path', done: true },
  { id: 1, text: 'Visit the temple', done: false },
  { id: 2, text: 'Drink matcha', done: false }
];

```
На следующем этапе вы удалите проход реквизита.
### Step 3: Use context anywhere in the tree [](https://react.dev/learn/scaling-up-with-reducer-and-context#step-3-use-context-anywhere-in-the-tree "Link for Step 3: Use context anywhere in the tree")
Теперь вам не нужно передавать список задач или обработчиков событий в дереве:
![[Pasted image 20240527212136.png]]
Вместо этого любой компонент, которому нужен список задач, может прочитать его из `TaskContext`:
![[Pasted image 20240527212150.png]]
Чтобы обновить список задач, любой компонент может прочитать функцию `dispatch` из контекста и вызвать ее:
![[Pasted image 20240527212203.png]]
Компонент TaskApp не передает никаких обработчиков событий, а TaskList также не передает никаких обработчиков событий компоненту Task. Каждый компонент считывает контекст, который ему нужен:

```js
import { useState, useContext } from 'react';
import { TasksContext, TasksDispatchContext } from './TasksContext.js';

export default function TaskList() {
  const tasks = useContext(TasksContext);
  return (
    <ul>
      {tasks.map(task => (
        <li key={task.id}>
          <Task task={task} />
        </li>
      ))}
    </ul>
  );
}

function Task({ task }) {
  const [isEditing, setIsEditing] = useState(false);
  const dispatch = useContext(TasksDispatchContext);
  let taskContent;
  if (isEditing) {
    taskContent = (
      <>
        <input
          value={task.text}
          onChange={e => {
            dispatch({
              type: 'changed',
              task: {
                ...task,
                text: e.target.value
              }
            });
          }} />
        <button onClick={() => setIsEditing(false)}>
          Save
        </button>
      </>
    );
  } else {
    taskContent = (
      <>
        {task.text}
        <button onClick={() => setIsEditing(true)}>
          Edit
        </button>
      </>
    );
  }
  return (
    <label>
      <input
        type="checkbox"
        checked={task.done}
        onChange={e => {
          dispatch({
            type: 'changed',
            task: {
              ...task,
              done: e.target.checked
            }
          });
        }}
      />
      {taskContent}
      <button onClick={() => {
        dispatch({
          type: 'deleted',
          id: task.id
        });
      }}>
        Delete
      </button>
    </label>
  );
}

```
Состояние по-прежнему «живет» в компоненте TaskApp верхнего уровня, управляемом с помощью useReducer. Но его задачи и диспетчеризация теперь доступны каждому компоненту ниже в дереве путем импорта и использования этих контекстов.
## Moving all wiring into a single file [](https://react.dev/learn/scaling-up-with-reducer-and-context#moving-all-wiring-into-a-single-file "Link for Moving all wiring into a single file")
Вам не обязательно этого делать, но вы можете еще больше навести порядок в компонентах, переместив и редуктор, и контекст в один файл. В настоящее время TasksContext.js содержит только два объявления контекста:
![[Pasted image 20240527212339.png]]
Этот файл скоро будет переполнен! Вы переместите редуктор в тот же файл. Затем вы объявите новый компонент TasksProvider в том же файле. Этот компонент свяжет все части вместе:
1. Он будет управлять состоянием с помощью редуктора.
2. Он предоставит оба контекста для компонентов ниже.
3. It will [take `children` as a prop](https://react.dev/learn/passing-props-to-a-component#passing-jsx-as-children) so you can pass JSX to it.
![[Pasted image 20240527212421.png]]
**Это устраняет всю сложность и проводку из вашего компонента `TaskApp`:**
```js
import AddTask from './AddTask.js';
import TaskList from './TaskList.js';
import { TasksProvider } from './TasksContext.js';

export default function TaskApp() {
  return (
    <TasksProvider>
      <h1>Day off in Kyoto</h1>
      <AddTask />
      <TaskList />
    </TasksProvider>
  );
}

```

Когда компоненту необходимо прочитать контекст, он может сделать это с помощью следующих функций:
![[Pasted image 20240527212514.png]]
Это никоим образом не меняет поведение, но позволяет позже разделить эти контексты или добавить некоторую логику к этим функциям. **Теперь весь контекст и проводка редуктора находится в `TasksContext.js`. Это сохраняет компоненты чистыми и незагроможденными, сосредоточившись на том, что они отображают, а не на том, где они получают данные:**
Вы можете думать о TasksProvider как о части экрана, которая знает, как обращаться с задачами, использовать Tasks как способ их чтения и использовать TasksDispatch как способ их обновления из любого компонента ниже в дереве.

Functions like `useTasks` and `useTasksDispatch` are called _[Custom Hooks.](https://react.dev/learn/reusing-logic-with-custom-hooks)_ Your function is considered a custom Hook if its name starts with `use`. This lets you use other Hooks, like `useContext`, inside it.
По мере роста вашего приложения у вас может быть много таких пар контекста и редуктора. Это мощный способ масштабировать ваше приложение и [поднять состояние](https://react.dev/learn/sharing-state-between-components) без особых работ, когда вы хотите получить доступ к данным глубоко в дереве.
