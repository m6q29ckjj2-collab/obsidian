Эта часть говорит о том, что функция вернет `true`, если тип `P` можно присвоить типу `T`, иначе `false`.
4. **Полный тип Equals**:
 ![[Pasted image 20240612141555.png]]
 В этой строке происходит сравнение двух функциональных типов:
 - Первый функциональный тип проверяет, является ли тип `P` подтипом `T`.
- Второй функциональный тип проверяет, является ли тип `P` подтипом `U`.
Если эти два функциональных типа эквивалентны, значит `T` и `U` эквивалентны, и тип `Equals<T, U>` будет равен `true`. В противном случае он будет равен `false`.

Вот как это работает на практике:
- Если `T` и `U` эквивалентны, оба функциональных типа будут одинаковыми, и `Equals<T, U>` будет `true`.
- Если `T` и `U` различны, функциональные типы будут различаться, и `Equals<T, U>` будет `false`

Основные компоненты
1. **Параметры типа**:
![[Pasted image 20240612151604.png]]
Здесь `T` - это массив, состоящий из элементов неизвестных типов (представленных как `unknown`), а `U` - это тип, который мы проверяем на наличие в массиве `T`.
2. **Mapped Type**:
![[Pasted image 20240612151644.png]]
Это создает новый тип, который маппирует каждый элемент массива `T` на результат проверки, эквивалентен ли этот элемент типу `U`.

- `keyof T` возвращает индексы массива `T` (то есть ключи массива).
- `T[K]` представляет тип элемента массива `T` по индексу `K`.
- `Equals<T[K], U>` проверяет, эквивалентен ли тип элемента массива `T` типу `U`.
**Индексный тип**:
![[Pasted image 20240612151756.png]]
Это извлекает тип всех значений из созданного mapped type. Например, если mapped type выглядит как `{ 0: false; 1: true; 2: false; }`, `[number]` извлечет `false | true | false`, то есть объединение всех значений.
4. **Условный тип**:
![[Pasted image 20240612151815.png]]
Этот условный тип проверяет, содержит ли объединение всех значений `false`. Если да, то результатом будет `false`, иначе - `true`.
