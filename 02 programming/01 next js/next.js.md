Включает в себя:
1) Routing - Маршрутизатор на основе файловой системы, построенный на основе серверных компонентов, который поддерживает макеты, вложенную маршрутизацию, состояния загрузки, обработку ошибок и многое другое.
2) Rendering - Клиентский и серверный рендеринг с использованием клиентских и серверных компонентов. Дальнейшая оптимизация с помощью статического и динамического рендеринга на сервере с помощью Next.js. Потоковая передача в среде выполнения Edge и Node.js.
3) Data fetching - Упрощенная выборка данных с помощью async/await в серверных компонентах и расширенный API выборки для запоминания запросов, кэширования и повторной проверки данных.
4) Styling - Поддержка предпочитаемых вами методов стилизации, включая модули CSS, Tailwind CSS и CSS-in-JS.
5) Optimizations - Оптимизация изображений, шрифтов и скриптов для улучшения основных веб-показателей и пользовательского опыта вашего приложения.
6) TypeScript - Улучшенная поддержка TypeScript с улучшенной проверкой типов и более эффективной компиляцией, а также специальным плагином TypeScript и средством проверки типов.
### App router vs pages router
В Next.js есть два разных маршрутизатора: маршрутизатор приложений и маршрутизатор страниц. App Router — это новый маршрутизатор, который позволяет использовать новейшие функции React, такие как серверные компоненты и потоковую передачу. Pages Router — это оригинальный маршрутизатор Next.js, который позволял создавать приложения React, отображаемые на сервере, и продолжает поддерживаться для более старых приложений Next.js.
