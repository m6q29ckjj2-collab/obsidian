Перехват маршрутов позволяет вам загрузить маршрут из другой части вашего приложения в текущем макете. Эта парадигма маршрутизации может быть полезна, если вы хотите отобразить содержимое маршрута без переключения пользователя на другой контекст.
For example, when clicking on a photo in a feed, you can display the photo in a modal, overlaying the feed. In this case, Next.js intercepts the `/photo/123` route, masks the URL, and overlays it over `/feed`
![[Pasted image 20240524182530.png]]
However, when navigating to the photo by clicking a shareable URL or by refreshing the page, the entire photo page should render instead of the modal. No route interception should occur.
![[Pasted image 20240524182601.png]]
## [Convention](https://nextjs.org/docs/app/building-your-application/routing/intercepting-routes#convention)
Перехватывающие маршруты можно определить с помощью соглашения (..), которое аналогично соглашению об относительном пути ../, но для сегментов.
You can use: 
 -  `(.)` to match segments on the **same level**
 -  `(..)` to match segments **one level above**
 - `(..)(..)` to match segments **two levels above**
 - `(...)` to match segments from the **root** `app` directory
 Например, вы можете перехватить сегмент фотографии из сегмента ленты, создав каталог фотографий (..).
![[Pasted image 20240524182841.png]]
Обратите внимание, что соглашение (..) основано на сегментах маршрута, а не на файловой системе.
## [Examples](https://nextjs.org/docs/app/building-your-application/routing/intercepting-routes#examples)
### [Modals](https://nextjs.org/docs/app/building-your-application/routing/intercepting-routes#modals)
Intercepting Routes can be used together with [Parallel Routes](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes) to create modals. This allows you to solve common challenges when building modals, such as:
- Создание модального контента, доступного через URL-адрес
- Сохранение контекста при обновлении страницы вместо закрытия модального окна.
- Закрытие модального окна при обратной навигации вместо перехода к предыдущему маршруту
- Повторное открытие модального окна прямой навигации

![[Pasted image 20240524183133.png]]
В приведенном выше примере путь к сегменту фотографии может использовать сопоставление (..), поскольку @modal — это слот, а не сегмент. Это означает, что маршрут фотографии находится всего на один уровень сегмента выше, несмотря на то, что он находится на два уровня файловой системы выше.

> **Good to know:**
> Другие примеры могут включать открытие модального окна входа в верхнюю панель навигации, а также наличие специальной страницы /login или открытие корзины покупок в боковом модальном окне.
