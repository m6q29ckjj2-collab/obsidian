Есть несколько способов обработки перенаправлений в Next.js. На этой странице будут рассмотрены все доступные опции, варианты использования и способы управления большим количеством перенаправлений.

## [`redirect` function](https://nextjs.org/docs/app/building-your-application/routing/redirecting#redirect-function)
Функция перенаправления позволяет перенаправить пользователя на другой URL-адрес. Вы можете вызвать перенаправление в компонентах сервера, обработчиках маршрутов и действиях сервера.

перенаправление часто используется после мутации или события. Например, создание публикации:
![[Pasted image 20240524141031.png]]
> **Good to know**:
> - redirect по умолчанию возвращает код состояния 307 (временное перенаправление). При использовании в действии сервера он возвращает код 303 (см. «Другое»), который обычно используется для перенаправления на страницу успеха в результате запроса POST.
> - Внутреннее перенаправление выдает ошибку, поэтому его следует вызывать вне блоков try/catch.
> - перенаправление можно вызвать в клиентских компонентах во время процесса рендеринга, но не в обработчиках событий. Вместо этого вы можете использовать хук useRouter.
> - redirect также принимает абсолютные URL-адреса и может использоваться для перенаправления на внешние ссылки.
> - Если вы хотите выполнить перенаправление до процесса рендеринга, используйте next.config.js или промежуточное ПО.

## [`permanentRedirect` function](https://nextjs.org/docs/app/building-your-application/routing/redirecting#permanentredirect-function)
Функция PermanentRedirect позволяет навсегда перенаправить пользователя на другой URL-адрес. Вы можете вызвать PermanentRedirect в компонентах сервера, обработчиках маршрутов и действиях сервера.

PermanentRedirect часто используется после мутации или события, которое меняет канонический URL-адрес объекта, например, при обновлении URL-адреса профиля пользователя после изменения имени пользователя:
![[Pasted image 20240524150832.png]]
> Good to know
>  - PermanentRedirect по умолчанию возвращает код состояния 308 (постоянное перенаправление).
>  - PermanentRedirect также принимает абсолютные URL-адреса и может использоваться для перенаправления на внешние ссылки.
>  - Если вы хотите выполнить перенаправление до процесса рендеринга, используйте next.config.js или middleware.

See the [`permanentRedirect` API reference](https://nextjs.org/docs/app/api-reference/functions/permanentRedirect) for more information.

## [`redirects` in `next.config.js`](https://nextjs.org/docs/app/building-your-application/routing/redirecting#redirects-in-nextconfigjs)
Параметр перенаправления в файле next.config.js позволяет перенаправить путь входящего запроса на другой путь назначения. Это полезно, когда вы меняете структуру URL-адресов страниц или имеете заранее известный список перенаправлений.
перенаправления поддерживают сопоставление путей, заголовков, файлов cookie и запросов, что дает вам возможность перенаправлять пользователей на основе входящего запроса.( [path](https://nextjs.org/docs/app/api-reference/next-config-js/redirects#path-matching), [header, cookie, and query matching](https://nextjs.org/docs/app/api-reference/next-config-js/redirects#header-cookie-and-query-matching))
To use `redirects`, add the option to your `next.config.js` file:
![[Pasted image 20240524151601.png]]
See the [`redirects` API reference](https://nextjs.org/docs/app/api-reference/next-config-js/redirects) for more information.
> **Good to know**:
> - перенаправления могут возвращать код состояния 307 (временное перенаправление) или 308 (постоянное перенаправление) с постоянной опцией.
> - перенаправления могут иметь ограничения на платформах. Например, на Vercel существует ограничение в 1024 перенаправления. Чтобы управлять большим количеством перенаправлений (более 1000), рассмотрите возможность создания собственного решения с использованием промежуточного программного обеспечения. > - перенаправления выполняются перед промежуточным программным обеспечением.

## [`NextResponse.redirect` in Middleware](https://nextjs.org/docs/app/building-your-application/routing/redirecting#nextresponseredirect-in-middleware)
Промежуточное программное обеспечение позволяет запускать код до завершения запроса. Затем на основе входящего запроса выполните перенаправление на другой URL-адрес с помощью NextResponse.redirect. Это полезно, если вы хотите перенаправлять пользователей на основе условия (например, аутентификация, управление сеансом и т. д.) или иметь большое количество перенаправлений.

## [Managing redirects at scale (advanced)](https://nextjs.org/docs/app/building-your-application/routing/redirecting#managing-redirects-at-scale-advanced)
Чтобы управлять большим количеством перенаправлений (более 1000), вы можете рассмотреть возможность создания собственного решения с использованием промежуточного программного обеспечения. Это позволяет вам программно обрабатывать перенаправления без необходимости повторного развертывания приложения.
Для этого вам необходимо учитывать:
1. Создание и сохранение карты перенаправления.
2. Оптимизация производительности поиска данных.
> **Next.js Example**: See our [Middleware with Bloom filter](https://redirects-bloom-filter.vercel.app/) example for an implementation of the recommendations below.

### [2. Optimizing data lookup performance](https://nextjs.org/docs/app/building-your-application/routing/redirecting#2-optimizing-data-lookup-performance)
Чтение большого набора данных для каждого входящего запроса может быть медленным и дорогостоящим. Существует два способа оптимизации производительности поиска данных:
- Используйте базу данных, оптимизированную для быстрого чтения, например Vercel Edge Config или Redis.
- Используйте стратегию поиска данных, например фильтр Блума, чтобы эффективно проверять наличие перенаправления перед чтением более крупного файла или базы данных перенаправлений.
Учитывая предыдущий пример, вы можете импортировать сгенерированный файл фильтра Блума в Middleware, а затем проверить, существует ли путь входящего запроса в фильтре Блума.
Если это так, перенаправьте запрос обработчику маршрута, который проверит фактический файл и перенаправит пользователя на соответствующий URL-адрес. Это позволяет избежать импорта большого файла перенаправлений в промежуточное ПО, что может замедлить каждый входящий запрос.
