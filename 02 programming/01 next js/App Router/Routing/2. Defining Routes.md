### Creating Routes
Next.js использует маршрутизатор на основе файловой системы, где папки используются для определения маршрутов.

Каждая папка представляет собой сегмент маршрута, который сопоставляется с сегментом URL-адреса. Чтобы создать вложенный маршрут, вы можете вкладывать папки друг в друга.
![[Pasted image 20240523212219.png]]
Специальный файл page.js используется для того, чтобы сделать сегменты маршрута общедоступными.
![[Pasted image 20240523212243.png]]
В этом примере URL-путь /dashboard/analytics не является общедоступным, поскольку у него нет соответствующего файла page.js. Эту папку можно использовать для хранения компонентов, таблиц стилей, изображений или других совместно расположенных файлов.

### Creating UI
Для создания пользовательского интерфейса для каждого сегмента маршрута используются специальные соглашения о файлах. Наиболее распространенными являются страницы, на которых отображается пользовательский интерфейс, уникальный для маршрута, и макеты, на которых отображается пользовательский интерфейс, общий для нескольких маршрутов.
