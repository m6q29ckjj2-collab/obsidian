Параллельные маршруты позволяют одновременно или условно отображать одну или несколько страниц в одном макете. Они полезны для высокодинамичных разделов приложения, таких как информационные панели и ленты социальных сайтов.

Например, рассматривая панель мониторинга, вы можете использовать параллельные маршруты для одновременной визуализации страниц команды и аналитики:

## [Slots](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#slots)
Параллельные маршруты создаются с использованием именованных слотов. Слоты определяются соглашением @folder. Например, следующая файловая структура определяет два слота: @analytics и @team:
![[Pasted image 20240524165307.png]]
Слоты передаются как реквизиты в общий родительский макет. В приведенном выше примере компонент в app/layout.js теперь принимает реквизиты слотов @analytics и @team и может отображать их параллельно вместе с реквизитом Children:
![[Pasted image 20240524165342.png]]
Однако слоты не являются сегментами маршрута и не влияют на структуру URL. Например, для /@analytics/views URL-адрес будет /views, поскольку @analytics — это слот.
> **Good to know**:
> Children prop — это неявный слот, который не нужно сопоставлять с папкой. Это означает, что app/page.js эквивалентен app/@children/page.js.

## [Active state and navigation](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#active-state-and-navigation)
По умолчанию Next.js отслеживает активное состояние (или подстраницу) для каждого слота. Однако контент, отображаемый в слоте, будет зависеть от типа навигации:
- Мягкая навигация: во время навигации на стороне клиента Next.js выполнит частичную визуализацию, изменяя подстраницу внутри слота, сохраняя при этом активные подстраницы другого слота, даже если они не соответствуют текущему URL-адресу.
- Сложная навигация: после полной загрузки страницы (обновления браузера) Next.js не может определить активное состояние для слотов, которые не соответствуют текущему URL-адресу. Вместо этого он отобразит файл default.js для несовпадающих слотов или 404, если default.js не существует.
> **Good to know**:
> Ошибка 404 для несовпадающих маршрутов помогает гарантировать, что вы случайно не отобразите параллельный маршрут на странице, для которой он не предназначен.

### [`default.js`](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#defaultjs)
Вы можете определить файл default.js для отображения в качестве резервного варианта для несовпадающих слотов во время начальной загрузки или полной перезагрузки страницы.
Рассмотрим следующую структуру папок. В слоте @team есть страница /settings, а в @analytics — нет.
![[Pasted image 20240524170223.png]]
При переходе к /settings слот @team отобразит страницу /settings, сохраняя при этом активную в данный момент страницу для слота @analytics.
При обновлении Next.js отобразит default.js для @analytics. Если default.js не существует, вместо него отображается ошибка 404.
Кроме того, поскольку Children — это неявный слот, вам также необходимо создать файл default.js для отрисовки резервного варианта для дочерних элементов, когда Next.js не может восстановить активное состояние родительской страницы.
### [`useSelectedLayoutSegment(s)`](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#useselectedlayoutsegments)
И useSelectedLayoutSegment, и useSelectedLayoutSegments принимают параметр ParallelRoutesKey, который позволяет вам читать активный сегмент маршрута внутри слота.
![[Pasted image 20240524171958.png]]
When a user navigates to `app/@auth/login` (or `/login` in the URL bar), `loginSegment`will be equal to the string `"login"`.

### [Modals](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#modals)
Параллельные маршруты можно использовать вместе с перехватывающими маршрутами для создания модальных окон. Это позволяет решать типичные проблемы при создании модальных окон, такие как:
- Создание модального контента, доступного через URL-адрес
- Сохранение контекста при обновлении страницы вместо закрытия модального окна
- Закрытие модального окна при обратной навигации вместо перехода к предыдущему маршруту
- Повторное открытие модального окна прямой навигации
Рассмотрим следующий шаблон пользовательского интерфейса, в котором пользователь может открыть модальное окно входа из макета с помощью навигации на стороне клиента или получить доступ к отдельной странице /login:
![[Pasted image 20240524173606.png]]
Чтобы реализовать этот шаблон, начните с создания маршрута /login, который отображает вашу главную страницу входа.
![[Pasted image 20240524173645.png]]
![[Pasted image 20240524173655.png]]
Затем внутри слота @auth добавьте файл default.js, который возвращает значение null. Это гарантирует, что модальное окно не будет отображаться, когда оно неактивно.
![[Pasted image 20240524173723.png]]
Внутри слота @auth перехватите маршрут /login, обновив папку /(.)login. Импортируйте компонент \<Modal> и его дочерние элементы в файл /(.)login/page.tsx:
![[Pasted image 20240524173802.png]]
> **Good to know:**
> - Соглашение, используемое для перехвата маршрута, например. (.), зависит от структуры вашей файловой системы. См. соглашение о перехватывающих маршрутах.
> - Отделив функциональность \<Modal> от модального содержимого (\<Login>), вы можете гарантировать, что любой контент внутри модального окна, например. формы являются компонентами сервера. Дополнительную информацию см. в разделе  [Interleaving Client and Server Components](https://nextjs.org/docs/app/building-your-application/rendering/composition-patterns#supported-pattern-passing-server-components-to-client-components-as-props).

#### [Opening the modal](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#opening-the-modal)
Теперь вы можете использовать маршрутизатор Next.js для открытия и закрытия модального окна. Это гарантирует правильное обновление URL-адреса при открытии модального окна и при навигации вперед и назад.
Чтобы открыть модальное окно, передайте слот @auth в качестве реквизита родительскому макету и визуализируйте его вместе с дочерним реквизитом.
![[Pasted image 20240524181450.png]]
Когда пользователь нажимает \<Link>, модальное окно открывается вместо перехода на страницу /login. Однако при обновлении или начальной загрузке переход к /login приведет пользователя на главную страницу входа.
#### [Closing the modal](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#closing-the-modal)
You can close the modal by calling `router.back()` or by using the `Link` component.
![[Pasted image 20240524181731.png]]
При использовании компонента Link для перехода со страницы, которая больше не должна отображать слот @auth, мы используем всеобъемлющий маршрут, который возвращает значение null.
![[Pasted image 20240524181812.png]]
> **Good to know:**
> - Мы используем универсальный маршрут в нашем слоте @auth, чтобы закрыть модальное окно из-за поведения, описанного в разделе «Активное состояние и навигация». Поскольку навигация на стороне клиента к маршруту, который больше не соответствует слоту, останется видимой, нам нужно сопоставить слот с маршрутом, который возвращает значение null, чтобы закрыть модальное окно.
> - Другие примеры могут включать открытие модального окна с фотографиями в галерее, а также наличие специальной страницы /photo/[id] или открытие корзины покупок в боковом модальном окне.
> - [View an example](https://github.com/vercel-labs/nextgram) of modals with Intercepted and Parallel Routes.
