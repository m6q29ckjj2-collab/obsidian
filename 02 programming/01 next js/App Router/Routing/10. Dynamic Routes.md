Если вы заранее не знаете точные имена сегментов и хотите создавать маршруты на основе динамических данных, вы можете использовать динамические сегменты, которые заполняются во время запроса или предварительно обрабатываются во время сборки.
## [Convention](https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes#convention)
Динамический сегмент можно создать, заключив имя папки в квадратные скобки: [имя_папки]. Например, [id] или [slug].
Динамические сегменты передаются в качестве свойства параметров для функций layout, page, route и generateMetadata.

## [Example](https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes#example)
Например, блог может включать следующий маршрут app/blog/[slug]/page.js, где [slug] — это динамический сегмент для сообщений в блоге.
![[Pasted image 20240524164320.png]]
See the [generateStaticParams()](https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes#generating-static-params) page to learn how to generate the params for the segment.
> **Good to know**
> Динамические сегменты эквивалентны динамическим маршрутам в каталоге страниц.

## [Generating Static Params](https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes#generating-static-params)
Функцию generStaticParams можно использовать в сочетании с сегментами динамического маршрута для статического создания маршрутов во время сборки, а не по требованию во время запроса.
![[Pasted image 20240524164523.png]]
Основное преимущество функции generateStaticParams — интеллектуальное извлечение данных. Если содержимое извлекается с помощью функции generateStaticParams с помощью запроса на выборку, запросы автоматически запоминаются. Это означает, что запрос на выборку с одинаковыми аргументами для нескольких GenerateStaticParams, Layouts и Pages будет выполнен только один раз, что сокращает время сборки.
Use the [migration guide](https://nextjs.org/docs/app/building-your-application/upgrading/app-router-migration#dynamic-paths-getstaticpaths) if you are migrating from the `pages` director
See [`generateStaticParams` server function documentation](https://nextjs.org/docs/app/api-reference/functions/generate-static-params) for more information and advanced use cases.
## [Catch-all Segments](https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)
Динамические сегменты можно расширить, чтобы охватить все последующие сегменты, добавив многоточие внутри скобок [...имя_папки].
For example, `app/shop/[...slug]/page.js` will match `/shop/clothes`, but also `/shop/clothes/tops`, `/shop/clothes/tops/t-shirts`, and so on.
![[Pasted image 20240524164733.png]]
## [Optional Catch-all Segments](https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes#optional-catch-all-segments)
Сегменты Catch-all можно сделать необязательными, включив параметр в двойные квадратные скобки: \[[...folderName]].
For example, `app/shop/[[...slug]]/page.js` will **also** match `/shop`, in addition to `/shop/clothes`, `/shop/clothes/tops`, `/shop/clothes/tops/t-shirts`
Разница между сегментами catch-all и необязательными сегментами catch-all заключается в том, что с необязательным также сопоставляется маршрут без параметра (/shop в примере выше).
