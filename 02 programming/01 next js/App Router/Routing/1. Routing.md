###  Routing Fundamentals
### Терминология 
![[Pasted image 20240523212940.png]]
- <mark style="background: #FFB8EBA6;">Tree</mark> - Соглашение о визуализации иерархической структуры. Например, дерево компонентов с родительскими и дочерними компонентами, структура папок и т. д.
- <mark style="background: #FFB8EBA6;">Subtree</mark> - Часть дерева, начинающаяся от нового корня (первая) и заканчивающаяся листьями (последняя).
- <mark style="background: #FFB8EBA6;">Root</mark> - Первый узел в дереве или поддереве, например корневой макет.
- <mark style="background: #FFB8EBA6;">Leaf</mark> - Узлы в поддереве, у которых нет дочерних элементов, например последний сегмент URL-пути.
![[Pasted image 20240523213000.png]]
- <mark style="background: #FFB8EBA6;">URL Segment</mark> - Часть URL-пути, разделенная косой чертой.
- <mark style="background: #FFB8EBA6;">URL Path</mark> - Часть URL-адреса, идущая после домена (состоящая из сегментов).
### The app Router
В версии 13 Next.js представлен новый маршрутизатор приложений, созданный на основе серверных компонентов React, который поддерживает общие макеты, вложенную маршрутизацию, состояния загрузки, обработку ошибок и многое другое.
Маршрутизатор приложений работает в новом каталоге с именем app. Каталог приложений работает вместе с каталогом страниц, обеспечивая постепенное внедрение. Это позволяет вам настроить некоторые маршруты вашего приложения на новое поведение, сохраняя при этом другие маршруты в каталоге страниц для предыдущего поведения. Если ваше приложение использует каталог страниц, см. также документацию Pages Router.

_Полезно знать: Маршрутизатор приложений имеет приоритет над Маршрутизатором страниц. Маршруты между каталогами не должны разрешаться по одному и тому же URL-пути и приведут к ошибке во время сборки, чтобы предотвратить конфликт._

По умолчанию компоненты внутри приложения — это компоненты React Server. Это оптимизация производительности, позволяющая легко их внедрить, а также использовать клиентские компоненты.

### Roles of Folders and Files
Next.js использует маршрутизатор на основе файловой системы, где:
- Папки используются для определения маршрутов. Маршрут — это единый путь вложенных папок, следующий по иерархии файловой системы от корневой папки до конечной папки, содержащей файл page.js. См. Определение маршрутов.
- Файлы используются для создания пользовательского интерфейса, отображаемого для сегмента маршрута. Посмотреть специальные файлы

### Route Segments 
Каждая папка в маршруте представляет собой сегмент маршрута. Каждый сегмент маршрута сопоставляется с соответствующим сегментом в пути URL.
![[Pasted image 20240523225032.png]]

### Nested Routes 
Чтобы создать вложенный маршрут, вы можете вкладывать папки друг в друга. Например, вы можете добавить новый маршрут /dashboard/settings, вложив две новые папки в каталог приложения.

- - `/` (Root segment)
- - `dashboard` (Segment)
- - `settings` (Leaf segment)

### Component Hierarchy
Компоненты React, определенные в специальных файлах сегмента маршрута, отображаются в определенной иерархии:
- `layout.js`
- `template.js`
- `error.js`
- `loading.js`
- `not-found.js`
- `page.js` or nested `layout.js`
![[Pasted image 20240523225616.png]]
Во вложенном маршруте компоненты сегмента будут вложены в компоненты его родительского сегмента.
![[Pasted image 20240523225659.png]]

###  [Colocation](https://nextjs.org/docs/app/building-your-application/routing#colocation)
Помимо специальных файлов, у вас есть возможность размещать свои собственные файлы (например, компоненты, стили, тесты и т. д.) внутри папок каталога приложения.
Это связано с тем, что, хотя папки определяют маршруты, публично адресуется только содержимое, возвращаемое page.js или Route.js.
![[Pasted image 20240523225916.png]]
### [Advanced Routing Patterns](https://nextjs.org/docs/app/building-your-application/routing#advanced-routing-patterns)
Маршрутизатор приложений также предоставляет набор соглашений, которые помогут вам реализовать более сложные шаблоны маршрутизации. К ним относятся:
- [[11. Parallel Routes]] - Позволяет одновременно отображать две или более страницы в одном представлении, по которым можно перемещаться независимо. Вы можете использовать их для разделенных представлений, имеющих собственную вложенную навигацию. Например. Панели мониторинга.
- [[Intercepting Routes]] - Позволяет перехватить маршрут и показать его в контексте другого маршрута. Вы можете использовать их, когда важно сохранить контекст текущей страницы. Например. Просмотр всех задач при редактировании одной задачи или расширении фотографии в ленте.
