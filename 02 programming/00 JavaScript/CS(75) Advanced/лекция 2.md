## Objects
 > [!question] 
> Какие существуют преобразования?
> 

> [!success] 
> В число или строку(примитивы)
> 

ToBoolean() не изменяет тип, а просто выводит true или false

То что нам дают в ToString или ToNumber сначала преобразовать в примитив.

String({... }) -> ToString({...}) -> ToPrimitive(string( <mark style="background: #FF5582A6;">hint</mark> )) ->  
ToPrimitive(хинт)  - объект у себя ищет метод, который переводит в текст или число, смотря от подсказки.

Number({...}) -> ToNumber({...}) ->ToPrimitive('number')

<mark style="background: #FF5582A6;">3 hint :</mark>
1. string (намёк - Конкретная операция, например alert() ожидает строку  - <mark style="background: #ADCCFFA6;">неявное преобразование</mark> , String({...}))
2. number(Number(), +{} - перевестись в ToNumber, {} - 3, {} > 3)
3. default(либо текст, либо в набер, объект должен сам решить.Когда оператор не уверен какой тип ожидать, например "+", ==  )

ToPrimitive(string)
```js
var hello = {...}
// String(hello)
// ToString(hello)
//ToPrimitive("string")
```

Метод в объекте:
1) hello.toDo() 
2) hello\\\["toDo"]()
> [!danger] 
> Методы можно было бы переписать

Для чего нужен <mark style="background: #FF5582A6;">Symbol</mark>?
hello\[Symbol.ToPrimitive](hint)

Так можно задать св-во, которое потом будет <mark style="background: #FFF3A3A6;">невозможно</mark> перезаписать 

Symbol.ToPrimitive - хранит какое-то значение, которое невозможно скопировать. 

Вызовы, если нет предыдущего у hello
1)  hello\\\[Symbol.toPrimitive]("string")
2) hello.toString()
3) hello.valueOf()
4) Error
> [!help] 
> Как сделать так, чтобы создаваемый объект не был связан с родителем?
