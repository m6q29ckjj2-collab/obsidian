**Тайминги навигации (Navigation timings)** - это показатели, указывающие временные метки, в которые произошли события навигации. **Тайминги ресурсов (Resource timings)** - это детальные показатели по времени загрузки ресурсов.

В этой статье мы рассмотрим как [Performance Timing API](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming "В настоящее время эта страница доступна только на английском языке"), так и Performance Entry API. И хотя первый API считается устаревшим, он все ещё поддерживается всеми браузерами, он прост и о нем полезно знать. В свою очередь, Performance Entry API является более продвинутым инструментом, который позволяет не только получить более сложные данные, но и позволяет разработчику измерять другие показатели, в дополнение к данным о навигации и загрузке ресурсов.

## [**Performance Timing** API](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#performance_timing_api)
[PerformanceTiming API](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming "В настоящее время эта страница доступна только на английском языке") - это JavaScript API для измерения времени загрузки страницы. Этот API считается устаревшим, но поддерживается во всех браузерах. На текущий момент рекомендуется использовать [performanceNavigationTiming](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceNavigationTiming "В настоящее время эта страница доступна только на английском языке") API.

[PerformanceTiming API](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming "В настоящее время эта страница доступна только на английском языке") предоставляет собой read only данные в виде объекта, где значениями полей являются числа, указывающие на количество миллисекунд, которые прошли к моменту срабатывания того или иного события. настоящее время эта страница доступна только на английском языке"), [`unloadEventStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/unloadEventStart "В настоящее время эта страница доступна только на английском языке"), [`unloadEventEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/unloadEventEnd "В настоящее время эта страница доступна только на английском языке"), [`redirectStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/redirectStart "В настоящее время эта страница доступна только на английском языке"), [`redirectEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/redirectEnd "В настоящее время эта страница доступна только на английском языке"), [`fetchStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/fetchStart "В настоящее время эта страница доступна только на английском языке"), [`domainLookupStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domainLookupStart "В настоящее время эта страница доступна только на английском языке"), [`domainLookupEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domainLookupEnd "В настоящее время эта страница доступна только на английском языке"), [`connectStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/connectStart "В настоящее время эта страница доступна только на английском языке"), [`connectEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/connectEnd "В настоящее время эта страница доступна только на английском языке"), [`secureConnectionStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/secureConnectionStart "В настоящее время эта страница доступна только на английском языке"), [`requestStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/requestStart "В настоящее время эта страница доступна только на английском языке"), [`responseStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/responseStart "В настоящее время эта страница доступна только на английском языке"), [`responseEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/responseEnd "В настоящее время эта страница доступна только на английском языке"), [`domLoading`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domLoading "В настоящее время эта страница доступна только на английском языке"), [`loadEventStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/loadEventStart "В настоящее время эта страница доступна только на английском языке"), и [`loadEventEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/loadEventEnd "В настоящее время эта страница доступна только на английском языке").

Описание показателей:
- [`navigationStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/navigationStart "В настоящее время эта страница доступна только на английском языке") - Момент, когда предыдущий документ в том же контексте (табе) запускает событие unload. Если предыдущего документа не было, значение этого показателя будет таким же, как и `PerformanceTiming.fetchStart`.
- [`secureConnectionStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/secureConnectionStart "В настоящее время эта страница доступна только на английском языке") - Началась установка (handshake) безопасного соединения. Если безопасного соединения нет, то значение равно `0`
- [`redirectStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/redirectStart "В настоящее время эта страница доступна только на английском языке") - Последний HTTP редирект завершён, то есть последний байт HTTP-ответа был получен. Если никаких редиректов не было, или один из редиректов перевёл документ на другой origin, то значение равно `0`.
- [`connectStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/connectStart "В настоящее время эта страница доступна только на английском языке") - Запрос на открытие соединения отправлен в сеть. Если транспортный уровень модели OSI сообщает об ошибке и установка соединения запускаются заново, то возвращается время старта последней попытки соединения. Если используется постоянное соединение, то значение показателя будет таким же, как и `PerformanceTiming.fetchStart`.
- [`connectEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/connectEnd "В настоящее время эта страница доступна только на английском языке") - Момент, когда соединение открыто для передачи данных. Если транспортный уровень модели OSI сообщает об ошибке и установка соединения запускаются заново, то возвращается время завершения последней попытки соединения. Если используется постоянное соединение, то значение показателя будет таким же, как и `PerformanceTiming.fetchStart`. Соединение считается открытым, когда завершены все этапы установление безопасного соединения, например TLS Handshake или SOCKS Authentication.
- [`domainLookupEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domainLookupEnd "В настоящее время эта страница доступна только на английском языке") - Поиск домена завершён. Если используется постоянное соединение, или используются данные, сохранённые в локальном кеше, то значение показателя будет таким же, как и `PerformanceTiming.fetchStart`.
- [`domainLookupStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domainLookupStart "В настоящее время эта страница доступна только на английском языке") - Начался поиск домена. Если используется постоянное соединение, или используются данные, сохранённые в локальном кеше, то значение показателя будет таким же, как и `PerformanceTiming.fetchStart`.
- [`fetchStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/fetchStart "В настоящее время эта страница доступна только на английском языке") - Браузер готов к загрузке документа с помощью HTTP-запроса. Этот этап всегда **срабатывает до проверки**кеша приложения.
- [`requestStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/requestStart "В настоящее время эта страница доступна только на английском языке") - Браузер посылает запрос на получение документа с сервера или из кеша. Если транспортный уровень сообщает об ошибке отправки запроса, а соединение переоткрывается - этот показатель будет перезаписан данными нового запроса.
- [`responseStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/responseStart "В настоящее время эта страница доступна только на английском языке")- Браузер получает первый байт ответа от сервера, кеша или локального ресурса
- [`responseEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/responseEnd "В настоящее время эта страница доступна только на английском языке") -Браузер получает последний байт ответа от сервера, кеша или локального ресурса. Если соединение закрывается раньше получения последнего байта - значение параметра указывает на момент закрытия соединения.
- [`domLoading`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domLoading "В настоящее время эта страница доступна только на английском языке") - Парсер HTML начинает работу. В этот момент [`Document.readyState`](https://developer.mozilla.org/ru/docs/Web/API/Document/readyState) изменяется на `'loading'` и срабатывает событие [`readystatechange`](https://developer.mozilla.org/ru/docs/Web/API/Document/readystatechange_event "readystatechange").
- [`unloadEventStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/unloadEventStart "В настоящее время эта страница доступна только на английском языке")- Срабатывает событие [`unload`](https://developer.mozilla.org/ru/docs/Web/API/Window/unload_event "unload")>, что говорит о времени, когда предыдущий документ начал выгружаться. Если предыдущего документа не было или переход к текущей странице подразумевал изменение origin (в т.ч. из-за редиректов), значение параметра равно `0`.
- [`unloadEventEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/unloadEventEnd "В настоящее время эта страница доступна только на английском языке") - Обработчик события [`unload`](https://developer.mozilla.org/en-US/docs/Web/API/Window/unload_event "В настоящее время эта страница доступна только на английском языке") завершил свою работу. Если предыдущего документа не было или переход к текущей странице подразумевал изменение origin (в т.ч. из-за редиректов), значение параметра равно `0`.
- [`domInteractive`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domInteractive "В настоящее время эта страница доступна только на английском языке") - HTML парсер завершил работу над основным документом. В этот момент [`Document.readyState`](https://developer.mozilla.org/ru/docs/Web/API/Document/readyState)изменяется на `'interactive'` и срабатывает событие [`readystatechange`](https://developer.mozilla.org/ru/docs/Web/API/Document/readystatechange_event)
- [`domContentLoadedEventStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domContentLoadedEventStart "В настоящее время эта страница доступна только на английском языке") - Момент сразу перед тем, как парсер запускает событие [`DOMContentLoaded`](https://developer.mozilla.org/en-US/docs/Web/API/Document/DOMContentLoaded_event "В настоящее время эта страница доступна только на английском языке"). Это событие запускается после того, как все скрипты, которые должны исполниться сразу после парсинга, выполнены.
- [`domContentLoadedEventEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domContentLoadedEventEnd "В настоящее время эта страница доступна только на английском языке") - Момент сразу после исполнения всех скриптов, которые должны были исполниться.
- [`domComplete`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/domComplete "В настоящее время эта страница доступна только на английском языке") - Парсер HTML завершил работу над основным документом. В этот момент [`Document.readyState`](https://developer.mozilla.org/ru/docs/Web/API/Document/readyState)изменяется на `'complete'` и срабатывает событие [`readystatechange`](https://developer.mozilla.org/ru/docs/Web/API/Document/readystatechange_event).
- [`loadEventStart`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/loadEventStart "В настоящее время эта страница доступна только на английском языке") - Событие [`load`](https://developer.mozilla.org/ru/docs/Web/API/Window/load_event) было отправлено текущему документу. Если это событие на момент измерения не было отправлено документу, значение параметра равно `0.`
- [`loadEventEnd`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceTiming/loadEventEnd "В настоящее время эта страница доступна только на английском языке") - Обработка события [`load`](https://developer.mozilla.org/ru/docs/Web/API/Window/load_event) завершена, то есть загрузка завершена. Если это событие ещё не произошло или не было послано документу, значение параметра равно `0`.

## [Performance Entry API](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#performance_entry_api)
Основные показатели производительности, рассмотренные выше, считаются устаревшими, но полностью поддерживаются современными браузерами. Взамен предлагается использовать [`Performance Entry API`](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceEntry "В настоящее время эта страница доступна только на английском языке"), который предоставляет инструмент для пометок и измерений времени одновременно с событиями navigation и загрузкой resource. ## [Navigation](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#navigation)
Когда пользователь запрашивает веб-приложение, [браузер должен получить некоторые мета-данные](https://developer.mozilla.org/en-US/docs/Web/Performance/How_browsers_work "В настоящее время эта страница доступна только на английском языке"), чтобы начать загрузку. Для этого пользовательский агент проходит серию шагов, такие как поиск записи DNS ([DNS](https://developer.mozilla.org/ru/docs/Glossary/DNS) lookup), TCP рукопожатие [TCP handshake](https://developer.mozilla.org/en-US/docs/Glossary/TCP_handshake "В настоящее время эта страница доступна только на английском языке"), и установку безопасного соединения (SSL negotiation). Как только браузер установил соединение, происходит первый полезный запрос данных на сервера. Как только начинают поступать данные от сервера, браузер начинает парсить полученные данные, строит DOM, CSSOM, создаёт деревья рендера (render trees), чтобы в конце концов отрендерить страницу. В тот момент, когда браузер перестаёт парсить входящие данные, документ переходит в интерактивную стадию. Если в документе существуют отложенные к загрузке ресурсы (deferred scripts), которые должны быть обработаны, браузер парсит их. После этого запускается событие [DOMContentLoaded](https://developer.mozilla.org/ru/docs/Web), после которого готовность страницы завершена. Теперь документ может обрабатывать пост-загрузочные задачи. После этого документ маркируется, как полностью загруженный.

Обратите внимание, что это данные для одного единственно документа, а не для всех ресурсов вместе взятых. В то же время, длительность загрузки, события-обработчики и тайминги построения DOM / CSSOM влияют на продолжительность загрузки всего приложения, не только одного конкретного ресурса. Клиентские приложения, выполняющиеся в браузере, могут выглядеть быстрее, если данные объёмом 300КБ вы передаёте сжатыми до 100КБ, но это все не значит, что JavaScript, CSS или другие медиа-ресурсы не раздувают приложение и не делают его медленнее. Проверка уровня сжатия - это очень важно, но не менее важно проверять длительность парсинга ресурсов и время между тем, как завершён DOMContentLoaded и DOM готов к работе. Может случиться так, что время парсинга скриптов и обработка скриптами результатов в основном потоке (main thread) приведёт к зависанию интерфейса.
### [Время запроса](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#%D0%B2%D1%80%D0%B5%D0%BC%D1%8F_%D0%B7%D0%B0%D0%BF%D1%80%D0%BE%D1%81%D0%B0)
API не предоставляет все измерения, которые разработчик хочет получить. Например, как долго продлилось выполнение запроса? Отдельного поля в объекте данных нет. Однако, мы можем использовать измерения, чтобы вычислить то, что нам нужно.

Чтобы определить время ответа, вычтите время старта запроса из времени старта получения ответа. Запрос стартует ровно в тот момент, когда клиент запрашивает ресурс с сервера (или из кеша). Ответ начинается ровно в тот момент, когда клиент получает первый байт.

### [DOMContentLoaded event](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#domcontentloaded_event)
Длительность события DOMContentLoaded определяется разностью моментов, когда клиент запускает событие DOMContentLoaded и когда это событие завершено. Старайтесь держать эту величину меньше 50ms - тогда ваш интерфейс будет отзывчивым.

### [Длительность (Duration)](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#%D0%B4%D0%BB%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D0%BE%D1%81%D1%82%D1%8C_duration)
В объекте данных есть поле Длительность (`Duration`). Длительность - это разница между [PerformanceNavigationTiming.loadEventEnd](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceNavigationTiming/loadEventEnd "В настоящее время эта страница доступна только на английском языке") и [PerformanceEntry.startTime](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceEntry/startTime "В настоящее время эта страница доступна только на английском языке") properties.

Интерфейс PerformanceNavigationTiming, кроме того, даёт информацию о том, какой тип навигации вы измеряете, возвращая `navigate`, `reload`, `back_forward` или `prerender`.

## [Resource](https://developer.mozilla.org/ru/docs/Web/Performance/Navigation_and_resource_timings#resource)
В то время, как тайминги навигации измеряют производительность загрузки и парсинга основного файла HTML, этот файл служит лишь точкой входа для загрузки других ресурсов. Поэтому нам так же важно знать, как быстро загружаются дополнительные ресурсы. Для измерения этих данных нужно использовать Resource Timing. Большая часть измерений в этом объекте похожи: здесь и поиск домена в DNS, и TCP установка соединения и т.д.
