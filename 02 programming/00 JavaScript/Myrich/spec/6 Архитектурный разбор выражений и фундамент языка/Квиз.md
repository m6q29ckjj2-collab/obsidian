> [!question] Рассмотрим выполнение следующего кода в браузере:
> ```js
> console.log(typeof setTimeout);
> ```
> Какое утверждение наиболее точно объясняет происхождение идентификатора `setTimeout`?
> a) Он является частью спецификации ECMAScript и создаётся Агентом.
> b) Его создаёт Host, предоставляя Binding в глобальном окружении до начала выполнения пользовательского кода.
> c) Он появляется только после создания первого Execution Context.
> d) Он автоматически наследуется через `Object.prototype`.
>
>> [!success]- Answer
>> b) Его создаёт Host, предоставляя Binding в глобальном окружении до начала выполнения пользовательского кода.
>> Explanation:
>> Спецификация ECMAScript не определяет `setTimeout`; его предоставляет Host. Binding создаётся до выполнения пользовательского кода.
>> Вариант a неверен, потому что `setTimeout` отсутствует в ECMAScript.
>> Вариант c неверен: Execution Context использует уже подготовленное окружение.
>> Вариант d неверен: поиск идентификатора не связан с прототипной цепочкой.
