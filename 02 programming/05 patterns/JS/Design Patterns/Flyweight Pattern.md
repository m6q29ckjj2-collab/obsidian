Шаблон «приспособленец» — полезный способ сэкономить память при создании большого количества похожих объектов.

В нашем приложении мы хотим, чтобы пользователи могли добавлять книги. У всех книг есть название, автор и номер isbn! Однако в библиотеке обычно имеется не один экземпляр книги: обычно имеется несколько экземпляров одной и той же книги.

Было бы не очень полезно каждый раз создавать новый экземпляр книги, если существует несколько копий одной и той же книги. Вместо этого мы хотим создать несколько экземпляров конструктора Book, которые представляют одну книгу.

![[Pasted image 20240714121550.png]]
Давайте создадим функционал для добавления новых книг в список. Если книга имеет тот же номер ISBN и, следовательно, является книгой того же типа, мы не хотим создавать совершенно новый экземпляр книги. Вместо этого нам следует сначала проверить, существует ли уже эта книга.

![[Pasted image 20240714121614.png]]
Если она еще не содержит номера ISBN книги, мы создадим новую книгу и добавим ее номер ISBN в набор isbnNumbers.
![[Pasted image 20240714121641.png]]
Функция createBook помогает нам создавать новые экземпляры книг одного типа. Однако в библиотеке обычно хранится несколько экземпляров одной и той же книги! Давайте создадим функцию addBook, которая позволит нам добавлять несколько копий одной и той же книги. Он должен вызвать функцию createBook, которая возвращает либо вновь созданный экземпляр Book, либо возвращает уже существующий экземпляр.

Чтобы отслеживать общее количество копий, давайте создадим массив bookList, содержащий общее количество книг в библиотеке.

![[Pasted image 20240714122541.png]]
Идеальный! Вместо создания нового экземпляра Book каждый раз при добавлении копии мы можем эффективно использовать уже существующий экземпляр Book для этой конкретной копии. Давайте создадим 5 экземпляров 3 книг: «Гарри Поттер», «Убить пересмешника» и «Великий Гэтсби».
![[Pasted image 20240714122940.png]]
Хотя существует 5 экземпляров, у нас всего 3 экземпляра Книги!

Шаблон «Легкий вес» полезен, когда вы создаете огромное количество объектов, которые потенциально могут истощить всю доступную оперативную память. Это позволяет минимизировать объем потребляемой памяти.

В JavaScript мы можем легко решить эту проблему с помощью прототипного наследования. В настоящее время аппаратное обеспечение имеет ГБ ОЗУ, что делает структуру «легковеса» менее важной.
