С помощью фабричного шаблона мы можем использовать фабричные функции для создания новых объектов. Функция является фабричной, если она возвращает новый объект без использования ключевого слова new!

Допустим, нашему приложению нужно много пользователей. Мы можем создавать новых пользователей с помощью свойств firstName, LastName и email. Фабричная функция также добавляет свойство FullName к вновь созданному объекту, которое возвращает firstName и LastName.

![[Pasted image 20240714120438.png]]
Идеальный! Теперь мы можем легко создать несколько пользователей, вызвав функцию createUser.

Фабричный шаблон может быть полезен, если мы создаем относительно сложные и настраиваемые объекты. Может случиться так, что значения ключей и значений зависят от определенной среды или конфигурации. С помощью фабричного шаблона мы можем легко создавать новые объекты, содержащие пользовательские ключи и значения!

![[Pasted image 20240714120527.png]]
### Props
Фабричный шаблон полезен, когда нам нужно создать несколько небольших объектов с одинаковыми свойствами. Фабричная функция может легко вернуть пользовательский объект в зависимости от текущей среды или пользовательской конфигурации.

## Cons
В JavaScript шаблон фабрики — это не что иное, как функция, которая возвращает объект без использования ключевого слова new. Стрелочные функции ES6 позволяют нам создавать небольшие фабричные функции, которые каждый раз неявно возвращают объект.

> [!tldr] 
> Однако во многих случаях более эффективно использовать память, создавая каждый раз новые экземпляры вместо новых объектов.
