Шаблон посредника позволяет компонентам взаимодействовать друг с другом через центральную точку: посредник. Вместо того, чтобы напрямую разговаривать друг с другом, посредник получает запросы и отправляет их вперед! В JavaScript посредник часто представляет собой не что иное, как литерал объекта или функцию.

Эту модель можно сравнить с отношениями между авиадиспетчером и пилотом. Вместо того, чтобы пилоты разговаривали друг с другом напрямую, что, вероятно, в конечном итоге было бы довольно хаотичным, пилоты разговаривают с авиадиспетчером. Авиадиспетчер следит за тем, чтобы все самолеты получали необходимую им информацию для безопасного полета, не задев другие самолеты.

Хотя мы надеемся, что мы не управляем самолетами в JavaScript, нам часто приходится иметь дело с разнонаправленными данными между объектами. Взаимодействие между компонентами может оказаться довольно запутанным, если имеется большое количество компонентов.

![[Pasted image 20240714123325.png]]
Вместо того, чтобы позволить каждому объекту напрямую взаимодействовать с другими объектами, что приводит к отношениям «многие ко многим», запросы объекта обрабатываются посредником. Посредник обрабатывает этот запрос и отправляет его туда, где он должен быть.

![[Pasted image 20240714123504.png]]
Хороший вариант использования шаблона посредника — это чат! Пользователи в чате не будут разговаривать друг с другом напрямую. Вместо этого чат служит посредником между пользователями.
![[Pasted image 20240714123525.png]]
Мы можем создавать новых пользователей, которые подключаются к чату. У каждого экземпляра пользователя есть метод отправки, который мы можем использовать для отправки сообщений.
## Case Study
Express.js — популярная платформа серверов веб-приложений. Мы можем добавить обратные вызовы к определенным маршрутам, к которым может получить доступ пользователь.

Допустим, мы хотим добавить заголовок к запросу, если пользователь попадает в корень «/». Мы можем добавить этот заголовок в обратный вызов промежуточного программного обеспечения.
![[Pasted image 20240714123657.png]]
Следующий метод вызывает следующий обратный вызов в цикле запрос-ответ. Фактически мы создадим цепочку функций промежуточного программного обеспечения, которые находятся между запросом и ответом, или наоборот.

Давайте добавим еще одну функцию промежуточного программного обеспечения, которая проверяет, правильно ли добавлен тестовый заголовок. Изменение, добавленное предыдущей функцией промежуточного программного обеспечения, будет видно по всей цепочке.

![[Pasted image 20240714123850.png]]

Perfect! We can track and modify the request object all the way to the response through one or multiple middleware functions.

Каждый раз, когда пользователь достигает корневой конечной точки «/», будут вызываться два обратных вызова промежуточного программного обеспечения.

Шаблон промежуточного программного обеспечения позволяет нам упростить отношения «многие ко многим» между объектами, позволяя всем коммуникациям проходить через одну центральную точку.
