Миксин — это объект, который мы можем использовать для добавления многократно используемой функциональности к другому объекту или классу без использования наследования. Мы не можем использовать миксины сами по себе: их единственная цель — добавить функциональность объектам или классам без наследования.

Допустим, для нашего приложения нам нужно создать несколько собак. Однако у базовой собаки, которую мы создаем, нет никаких свойств, кроме свойства имени.

![[Pasted image 20240714124431.png]]

Собака должна иметь возможность делать больше, чем просто иметь имя. Он должен уметь лаять, вилять хвостом и играть! Вместо того, чтобы добавлять это непосредственно в Dog, мы можем создать миксин, который предоставит нам свойства bark, wagTail и play.

![[Pasted image 20240714124512.png]]

Мы можем добавить примесь DogFunctionality к прототипу Dog с помощью метода Object.assign. Этот метод позволяет нам добавлять свойства к целевому объекту: в данном случае Dog.prototype. Каждый новый экземпляр Dog будет иметь доступ к свойствам DogFunctionality, поскольку они добавляются в прототип Dog!

![[Pasted image 20240714124550.png]]
Давайте создадим нашего первого питомца pet1 по имени Дейзи. Поскольку мы только что добавили миксин DogFunctionality к прототипу Dog, Дейзи должна уметь ходить, вилять хвостом и играть!
![[Pasted image 20240714124636.png]]
Идеальный! Миксины позволяют нам легко добавлять пользовательские функции к классам или объектам без использования наследования.

Хотя мы можем добавлять функциональность с помощью миксинов без наследования, сами миксины могут использовать наследование!

Большинство млекопитающих (кроме дельфинов... и, возможно, некоторых других) также могут ходить и спать. Собака — млекопитающее, и она должна уметь ходить и спать!

Давайте создадим миксин AnimalFunctionality, который добавляет свойства ходьбы и сна.
![[Pasted image 20240714124711.png]]
Мы можем добавить эти свойства в прототип DogFunctionality, используя Object.assign. В данном случае целевым объектом является DogFunctionality.
![[Pasted image 20240714125153.png]]

Пример миксина в реальном мире виден в интерфейсе Window в среде браузера. Объект Window реализует многие из своих свойств из миксинов WindowOrWorkerGlobalScope и WindowEventHandlers, которые позволяют нам иметь доступ к таким свойствам, как setTimeout и setInterval, indexedDB и isSecureContext.

Поскольку это миксин и используется только для добавления функциональности объектам, вы не сможете создавать объекты типа WindowOrWorkerGlobalScope.

## React (pre ES6)
Миксины часто использовались для добавления функциональности компонентам React до появления классов ES6. Команда React не рекомендует использовать примеси, поскольку они легко усложняют компонент, затрудняя его поддержку и повторное использование. Команда React рекомендовала вместо этого использовать компоненты более высокого порядка, которые теперь часто можно заменить хуками.

Миксины позволяют нам легко добавлять функциональность к объектам без наследования, внедряя функциональность в прототип объекта. Изменение прототипа объекта считается плохой практикой, поскольку оно может привести к загрязнению прототипа и повышению уровня неопределенности относительно происхождения наших функций.
