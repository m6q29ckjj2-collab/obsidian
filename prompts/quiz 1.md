# Цель

Проверить глубокое понимание материала, а не способность его запомнить.

# Требования к вопросам

Каждый вопрос должен требовать хотя бы одно из следующих действий:

- анализ ситуации;
- сравнение нескольких концепций;
- поиск причин и следствий;
- предсказание результата изменения условий;
- применение нескольких частей материала одновременно;
- выявление скрытой ошибки в рассуждении;
- выбор наиболее корректного объяснения.

Запрещено:

- спрашивать определения;
- спрашивать даты;
- спрашивать формулы без понимания их смысла;
- спрашивать термины;
- спрашивать факты, которые можно просто вспомнить из текста;
- перефразировать предложения из текста.

Если вопрос можно решить простой зубрежкой — перепиши его.

# Сложность

Представь, что экзамен принимает преподаватель университета, который хочет отличить человека, действительно понимающего тему, от человека, который просто прочитал конспект.

Неправильные ответы должны быть максимально правдоподобными.

Используй типичные ошибки мышления опытных студентов.

Не делай очевидно абсурдных вариантов.

# Покрытие материала

Создай 20 вопросов.

Распредели их максимально равномерно по всем разделам текста.

Не допускай повторения одной и той же идеи в разных вопросах.

Минимум половина вопросов должна объединять материал сразу из нескольких разделов.

Минимум треть вопросов должна использовать сценарии ("Что произойдет, если...", "Предположим...", "Исследователь обнаружил...", "В эксперименте...").

# Объяснения

После каждого вопроса обязательно:

- объясни, почему правильный ответ правильный;
- отдельно объясни, почему каждый неправильный вариант неверен;
- объяснения должны ссылаться на логику материала, а не просто повторять правильный ответ.

# Формат вывода

Выводи ТОЛЬКО вопросы в формате, полностью совместимом с Obsidian Quiz Generator.

Каждый вопрос должен иметь строго следующий вид:

> [!question] Текст вопроса
> a) Первый вариант
> b) Второй вариант
> c) Третий вариант
> d) Четвертый вариант
>
>> [!success]- Answer
>> b) Полный текст правильного ответа
>> Explanation:
>> Почему правильный ответ верен.
>> Почему вариант a неверен.
>> Почему вариант c неверен.
>> Почему вариант d неверен.

- Каждый вопрос начинается с `> [!question]`.

- Ответ начинается только с `>> [!success]- Answer`.
- После ответа обязательно идет строка `>> Explanation:`.
- Каждая строка объяснения начинается с `>>`.
- В правильном ответе указывай букву и полный текст ответа.
- Между вопросами оставляй ровно одну пустую строку.
- Не используй заголовки.
- Не используй Markdown кроме указанного.
- Не используй YAML.
- Не используй HTML.
- Не используй разделители.
- Не добавляй вступление.
- Не добавляй заключение.
- Верни только готовый квиз.
